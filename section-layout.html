Layout

Components used for page layout

Grids

Used for creating a columned layout. To use these classes, just pick the number of columns wide you want your element to be relative to the total number of columns. The default is 12 columns. For example, you can use .grid--4 to make an element 4 columns wide followed by an element with .grid--8 to take up the remaining eight columns of the container's width.

Example

.grid--4
.grid--4
.grid--4
.grid--2
.grid--7
.grid--3

Table Grids

Simple helper classes to be used on a table to set column widths based on the same 12 column grid system used in the Grid component. THese classes are most useful when it is necessary to specikfy the width of table columns. One example would be if there are several tables on the same page with a similar structure and each tables' columns should be uniform in size.

These classes can be used in virtually the same way as the Grid component except that they are either applied to table heading cells (<th>) or table row cells (<td>). In most cases these should just be applied once to each <th>, but in certain systems such as Drupal views, you might be limited to applying classes to row cells. This will work in the same way, but just produces slightly more markup as the classes get repeated for each row in the table.

Use classes table-grid--1 through table-grid--12 and make sure they always add up to 12 columns across a row.

Marginless Boxes

Percentage width boxes that butt up against eachother with no margin. Typically, a colored background would be applied with a .bg--{color} class. In a mobile layout, all stretchy boxes default to stacking one ontop of the other and in a tablet layout stretchy boxes default to a 50% split. A .box__stretchy--tablet-full class can be applied to a box to force it to be full-width on a tablet. This is useful when there are an odd number of boxes and you want to fill the extra blank space.
